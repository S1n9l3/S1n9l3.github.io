[ctf-reverse] CrackRTF

CrackRTF

下载文件是一个没加壳的exe文件，用IDA打开，找到关键的反汇编代码，

首先需要输入一个六位的数字，然后拼接为??????@DBApp，然后进入sub_40100A函数，返回结果是6E32D0943418C2C33385BC35A1470250DD8923A9，可能是某种加密算法。

sub_40100A函数

看函数应该是进行了加密，将几个函数在windowsAPI中查询，找到一个关键的函数CryptCreateHash，用于创建数据流哈希，用于后续调用CryptHashData，这个函数的第二个参数，表示了加密算法，这里的0x8004表示SHA1加密。

CryptCreateHash 函数启动数据流的哈希。 它创建并返回到调用应用程序，该句柄指向 加密服务提供程序 (CSP) 哈希对象。 此句柄用于后续调用 CryptHashData 和 CryptHashSessionKey 以哈希会话键和其他数据流。

BOOL CryptCreateHash(
[in] HCRYPTPROV hProv,
[in] ALG_ID Algid,
[in] HCRYPTKEY hKey,
[in] DWORD dwFlags,
[out] HCRYPTHASH *phHash
);

ALG_ID (Wincrypt.h) - Win32 apps | Microsoft Learn

所以可以推断这个函数就是对输入进行了sha1加密，既然是六位数字，可以尝试爆破；

这样就解出第一个密码，结果是123321。

然后要输入第二个密码，同样是六位的，但是不限于数字，将第一个字符串与第二个密码拼接??????123321@DBApp，然后进入sub_401019函数，结果是27019e688a4e62a649fd99cadaafdb4e，同样是加密。

sub_401019函数

结构一样，打算加密算法不同，这里是0x8003，表示MD5算法，但是范围太大了，不能爆破。

继续回到主函数，接着有个处理，

Str表示第二次拼接后的字符串，经过sub_40100F函数处理后，可得到结果，接着看这个函数，

如果这个函数返回1就可以了。

这里有几个函数：

  • FindResourceA 函数，确定具有指定模块中指定类型和名称的资源的位置。
  • SizeofResource函数，检索指定资源的大小（以字节为单位）。
  • LoadResource 函数，检索可用于获取指向内存中指定资源的第一个字节的指针的句柄。
  • LockResource 函数，检索指向内存中指定资源的指针。
  • CreateFileA 函数，创建或打开文件或 I/O 设备。
  • WriteFile 函数，将数据写入指定的文件或输入/输出 (I/O) 设备。

所以首先就是读取资源文件，用ResourceHacker可以查看exe中的资源文件，

还有一个函数sub_401005，

主要是进行了异或操作，读取资源文件AAA后，与第二个密码字符串进行了异或；

异或之后，创建文件dbapp.rtf，然后将结果写入文件，所以rft文件的前几个字节就是异或的结果，创建一个rtf文件查看其文件头标识，

取前六位就是{\rtf1，然后与资源AAA的前六位异或，

最后得到结果~!3a@0。

得到两个密码之后，就可以运行源程序，输入两次密码，然后会创建dbapp.rtf文件，得到flag。
